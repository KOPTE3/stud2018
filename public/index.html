Аудит безопасности

Аудит информационной безопасности — системный процесс получения объективных качественных и количественных оценок о текущем состоянии информационной безопасности компании в соответствии с определёнными критериями и показателями безопасности

Аудит позволяет оценить текущую безопасность функционирования информационной системы, оценить и прогнозировать риски, управлять их влиянием на бизнес-процессы фирмы, корректно и обоснованно подойти к вопросу обеспечения безопасности её информационных активов

Актуальность

  • Популяризация Интернета
  • Развитие возможностей web-приложений, которые теперь позволяют делать всё более сложные web-приложения
  • Появление интернет-банкинга, интернет-магазинов, большое количество операций с деньгами, проводящихся через web-приложения
  • Увеличение количества персональных данных, хранящихся и обрабатываемых в web-приложениях

Уязвимости и атаки на web-приложения

  • Cross-Site Scripting (XSS) — тип атаки на web-приложения, заключающийся во внедрении в выдаваемую сервером страницу вредоносного кода (который будет выполнен на компьютере пользователя при открытии им этой страницы) и взаимодействии этого кода с web-сервером злоумышленника
  • SQL-injection — тип атаки на web-приложения, работающие с SQL-базами данных, основанный на внедрении в запрос произвольного SQL-кода, который затем будет выполнен на web-сервере
  • Access Control List (ACL) vulnerabilities — уязвимости, связанные с неправильной проверкой прав доступа пользователей, когда одни пользователи системы имеют возможность взаимодействовать с объектами, созданными другими пользователями системы
  • Множество других типов атак и уязвимостей

Цифры ДО

  • Команда информационной безопасности Почты — 5 человек
  • Каждый релиз — около 400 проверок, выполняющихся руками (это только тестирование web-интерфейса)
  • Тестирование безопасности итерации занимало два дня

Что мы сделали

  1. Оформили часть проверок безопасности в виде тест-кейсов, пригодных к автоматизации
  2. С помощью имеющейся инфраструктуры, написали по этим кейсам автоматизированные тесты
  3. Стали запускать эти тесты совместно с интеграционными тестами на всё web-приложение

Цифры после

  • Автоматизировали проверку XSS, ACL и Template Injection уязвимостей
  • 240 автоматизированных тестов безопасности
  • Сокращение времени тестирования итерации с двух дней до 4–5 часов

Программные средства автоматизации тестирования

  • Протокол W3C WebDriver — программный интерфейс для управления браузером
  • Selenium — это инструмент для автоматизации действий веб-браузера
  • WebDriverIO — реализация протокола WebDriver для управления браузером из Node.js
  • Jenkins CI — проект для обеспечения непрерывной интеграции

Методика проверки защищённости от XSS

  1. Авторизуемся в web-приложении подготовленным аккаунтом
  2. Заходим на форму написания письма, отправляем себе письмо с темой, содержащей опасную комбинацию спецсимволов с XSS
  3. Переходим во Входящие
  4. Открываем письмо с необходимой темой
  5. Проверяем, что внедрённый код не отработал и что все спецсимволы в теме экранированы

Методика проверки защищённости от ACL Vulnerabilities

  1. Авторизуемся в web-приложении подготовленным аккаунтом
  2. Открываем письмо с вложением, копируем URI вложения
  3. Авторизуемся под другим аккаунтом
  4. Пытаемся открыть ресурс с ранее скопированным URI
  5. Убеждаемся, что ресурс не открывается

Методика проверки защищённости от Template Injection

  1. Авторизуемся в web-приложении подготовленным аккаунтом
  2. Переходим на страницу настройки списка папок
  3. Создаём новую папку с потенциально опасным названием
  4. Переходим на страницу лога действий (её содержимое полностью генерируется сервером)
  5. Проверяем, что вредоносный код не выполнился

Убеждаемся в работоспособности тестов

Мутационное тестирование (мутационный анализ или мутация программ) — это метод тестирования программного обеспечения, который включает небольшие изменения кода программы. Если набор тестов не в состоянии обнаружить такие изменения, то он рассматривается как недостаточный
